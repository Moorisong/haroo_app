# 화면 구성 및 UI/UX

## 앱 화면 구조

```
/home        → 홈 화면 (연결 상태 확인)
/request     → 메시지 모드 신청
/send        → 메시지 작성 및 전송
/receive     → 메시지 수신 확인
/settings    → 설정
```

---

## /home — 홈

### 화면 목적
지금 나의 연결 상태와 오늘 메시지를 보낼 수 있는지 확인하는 화면

### 상태별 구성

#### 1. 연결 없음 (NONE)

| 항목 | 내용 |
|------|------|
| 메인 문구 | "오늘, 마음을 전할 사람이 있나요?" |
| 버튼 | 메시지 모드 신청하기 → /request |

#### 2. 신청 대기 - 발신자 (PENDING)

| 항목 | 내용 |
|------|------|
| 메인 문구 | "상대의 수락을 기다리고 있어요." |
| 보조 문구 | "메시지 모드가 활성화되면 하루에 한 번 메시지를 보낼 수 있어요." |
| 버튼 | 없음 |

#### 3. 신청 대기 - 수신자 (PENDING)

| 항목 | 내용 |
|------|------|
| 메인 문구 | "누군가 마음을 전하고 싶어 해요." |
| 보조 문구 | "허락하면 하루에 한 번 메시지를 받을 수 있어요." |
| 버튼 | 수락하기 / 거절하기 / 이 발신자 차단하기 |

#### 4. 메시지 모드 활성 (ACTIVE_PERIOD)

| 항목 | 내용 |
|------|------|
| 상태 문구 | "메시지 모드 진행 중" |
| 보조 정보 | 남은 기간 D-n, 오늘 메시지 전송 가능 여부 |
| 버튼 | 오늘의 메시지 보내기 (가능할 때만 활성) |

#### 5. 기간 종료 (EXPIRED)

| 항목 | 내용 |
|------|------|
| 메인 문구 | "메시지 모드가 종료되었어요." |
| 버튼 | 다시 신청하기 → /request |

### 내 ID 표시

| 항목 | 내용 |
|------|------|
| 라벨 | 내 ID |
| ID 값 | (예: haru_x9f3a2) |
| 마이크로카피 | "이 ID로만 메시지 모드를 신청할 수 있어요." |
| 액션 | 복사하기 버튼 → 탭 시 클립보드 복사 |
| 피드백 | "ID가 복사되었어요." |

## /request — 메시지 모드 신청

### 화면 목적
상대방이 메시지 모드 신청이 가능한 상태인지 확인하고 신청

### UI 구성

| 항목 | 내용 |
|------|------|
| 상단 안내 | "상대방의 ID를 입력해 주세요." |
| 보조 안내 | "상대방에게는 내 ID를 공유해야 신청을 받을 수 있어요." |
| 입력창 placeholder | "상대방 ID를 입력하세요" |
| 버튼 | 확인하기 |

### 상태 결과

#### 신청 가능
- 안내: "이 사용자에게 메시지 모드를 신청할 수 있어요."
- 기간 선택 영역 표시 (1일/3일)
- 버튼: 메시지 모드 신청하기

#### 신청 불가
- 안내: "지금은 메시지 모드를 신청할 수 없는 상태예요."
- 버튼 없음

## /send — 메시지 작성

### 화면 목적
하루에 한 번, 상대에게 전달할 단일 메시지 작성

### 메시지 입력

| 항목 | 내용 |
|------|------|
| 글자 수 제한 | 40자 |
| 마이크로카피 | "한 문장이면 충분해요. 오늘의 마음만 전해도요." |
| 안내 문구 | "이 메시지는 하루에 한 번만 보낼 수 있어요." |

### 버튼
- "메시지 보내기"

---

## /receive — 메시지 수신

### 화면 목적
부담 없이 메시지를 읽는 공간 (대화 아님)

### UI 구성

| 항목 | 내용 |
|------|------|
| 메시지 표시 | 수신된 단일 메시지 1개 표시 |
| 고정 안내 | "이 메시지는 읽기만 가능합니다. 답장은 요구하지 않아요." |
| 제약 | 답장 / 반응 / 수정 ❌ |

### 표시 방식 안내 영역
- 현재 표시 방식: 알림바 고정 / 홈 화면 위젯
- 터치 시 /settings 이동

### 차단 버튼
- 위치: 메시지 본문 하단
- 문구: "이 발신자 차단하기"
- 동작: ACTIVE_PERIOD → BLOCKED

---

## /settings — 설정

### 화면 목적
메시지 표시 방식 선택 및 기타 설정

### 표시 방식 선택

| 선택지 | 설명 |
|--------|------|
| 홈 화면 위젯 | 일상에 조용히 두는 메시지 |
| 알림바 고정 | 하루 동안 지우지 않겠다는 선택 |

안내 문구: "메시지는 선택한 방식으로만 표시돼요."

### 기타 설정
- 내 ID 확인 및 복사
- 차단 목록 관리
- 앱 버전 정보

---

## UX 원칙 요약

| 원칙 | 설명 |
|------|------|
| 대화 ❌ | 채팅이 아닌 단일 메시지 |
| 답장 ❌ | 수신자에게 답장 강요하지 않음 |
| 하루 1회 | 메시지는 하루에 한 번만 |
| 단일 메시지 | 히스토리 최소화 |
| 허락된 연결 | 양측 동의 기반 |
