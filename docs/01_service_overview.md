# 앱하루 서비스 개요

## 한 줄 정의

> 하루에 한 번, 상호 허가된 상대에게 **유료로 마음을 전달하는 단일 메시지 앱**

## 핵심 철학

앱하루의 연결은 대화를 위한 것이 아니라, **단 한 번의 메시지를 허락하는 절차**이다.

---

## 핵심 원칙 (변경 불가)

| 원칙 | 설명 |
|------|------|
| 하루 1회 메시지 | 메시지는 하루에 한 번만 전송 가능 |
| 단건 결제 | 메시지 모드 신청 시 단건 결제 |
| 비대화 구조 | 채팅 ❌ / 대화 ❌ / 히스토리 최소화 |
| 단일 연결 | 한 유저는 동시에 1명과만 연결 가능 |
| 허가 기반 | 연결은 "대화"가 아닌 단일 메시지 허가 |

---

## 기술 스택

### 앱 (Mobile)
- **프레임워크**: React Native (Expo)
- **패키지 위치**: `packages/app`

### 백엔드 (Server)
- **프레임워크**: Node.js + Express
- **데이터베이스**: MongoDB (Mongoose)
- **패키지 위치**: `packages/server`

### 웹 (Landing/Promotional)
- **목적**: 앱 설치 유도 랜딩 페이지
- **패키지 위치**: `packages/web`

### 푸시 알림
- FCM (Firebase Cloud Messaging)
- APNs (Apple Push Notification service)

---

## 수익 모델

### 가격 및 기간

| 가격 | 기간 | 철학 |
|------|------|------|
| 500원 | 1일 | 오늘 떠오른 마음을 단 한 번 전달 |
| 1,000원 | 3일 | 조금 더 깊은 마음, 이어지는 시간 표현 |

### UX/철학 포인트
- 가격 차이는 메시지 길이나 기능이 아닌 **마음의 무게**를 표현
- 기간 차이는 **마음의 이어짐**을 의미
- 반복 결제 유도나 메시지 히스토리 ❌

## MVP 범위

### 포함
- 연결 신청 / 수락
- 메시지 전송
- 단건 결제
- 푸시 알림 + 위젯

### 제외
- 채팅
- 메시지 히스토리
- 소셜 기능

---

## 수신 방식 설정

사용자가 선택 가능한 메시지 표시 방식:
- **위젯**: 일상에 조용히 두는 메시지
- **고정 알림**: 하루 동안 지우지 않겠다는 선택

### 원칙
- ❌ 메시지 받을 때마다 선택하게 하지 않음
- ✅ **사전 설정 → 자동 적용** 구조
